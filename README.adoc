== Purpose

This repository contains sanctioned entities from the Japan End-User List
published by the Ministry of Economy, Trade and Industry (METI).

== Source

* **Authority**: Japan Ministry of Economy, Trade and Industry (METI)
* **URL**: https://www.meti.go.jp/policy/anpo/english/law/doc/EndUserListE.html
* **Format**: PDF (requires manual conversion to structured data)
* **Update Frequency**: As published by METI
* **Country Code**: JP

== Legal Basis

The End-User List is maintained under Japan's Foreign Exchange and Foreign
Trade Act (FEFTA) and related export control regulations.

== Directory Structure

```
data-jp/
├── downloaded/        # Raw downloaded files (PDF, HTML)
├── processed/         # Processed YAML files (one per entity)
├── reference-docs/    # Reference documentation and converted files
└── README.adoc        # This file
```

== Data Format

Each processed entity is stored as a YAML file with the following structure:

[source,yaml]
----
id: JP-001
name: Example Entity
name_ja: 例の団体
entity_type: organization
addresses:
  - city: Tokyo
    country: JP
source_url: https://www.meti.go.jp/...
----

== Processing

The source PDF must be converted to HTML before running the parser script.

=== Converting PDF to HTML

Install `poppler-utils` (for `pdftohtml`) or use an online converter:

[source,shell]
----
# Using poppler-utils (macOS)
brew install poppler

# Convert PDF to HTML
pdftohtml -c -noframes reference-docs/20250131-jp-end-user-list.pdf reference-docs/20250131-jp-end-user-list.html
----

=== Running the Parser

After conversion, run the parser from the ammitto root:

[source,shell]
----
bundle exec ruby scripts/parse_jp_end_user_list.rb
----

This will generate YAML files in the `processed/` directory.

== Knowledge Graph Integration

This repository's processed YAML data feeds into the Ammitto knowledge graph.

When the harmonization process runs:

[source,shell]
----
# Auto-detect all data-* repos and harmonize
ammitto harmonize --scan --sources-dir /path/to/repos --output-dir ./api/v1
----

The jp data is transformed into:

* Individual JSON-LD nodes in `api/v1/node/entity/jp/{id}.jsonld`
* Sanction entries in `api/v1/node/entry/jp/{id}.jsonld`
* Lightweight search index entries
* Facet counts for filtering
