== Purpose

This repository stores downloaded and processed sanctions data from the
Government of Japan sanctions announcements.

The data is maintained by the Ammitto project and used by the `ammitto` Ruby
gem for querying international sanctions information.

== Legal Basis

Japan maintains several sanctions and restricted entity lists under the
Foreign Exchange and Foreign Trade Act (FEFTA):

* **End-User List** (輸出規制リスト) - Ministry of Economy, Trade and Industry (METI)
* **Asset Freeze List** (資産冻结等リスト) - Ministry of Finance (MOF)
* **Travel Ban List** (入国禁止リスト) - Ministry of Foreign Affairs (MOFA)

These are implemented through the Foreign Exchange and Foreign Trade Act
(昭和二十四年法律第二百二十八号) and various cabinet orders and ministerial ordinances.

== Data Source

Japan maintains multiple sanctions lists:

.End-User List (輸出規制リスト)
* **Authority**: Ministry of Economy, Trade and Industry (経済産業省)
* **URL**: https://www.meti.go.jp/policy/anpo/
* **Effects**: Export license requirement, asset freeze

.Asset Freeze List (資産冻结等リスト)
* **Authority**: Ministry of Finance (財務省)
* **Effects**: Asset freeze, transaction ban

.Travel Ban List (入国禁止リスト)
* **Authority**: Ministry of Foreign Affairs (外務省)
* **Effects**: Entry ban, visa ban

=== Schema Types

Each YAML file follows one of two schema types:

. **Announcement Schema** - Standard sanction announcements with `sanction_details`
. **Legal Instrument Schema** - Laws and regulations with structured content

Schema files are located in `schemas/`:

* `schemas/jp-announcement.yml` - For sanction list announcements
* `schemas/jp-legal-instrument.yml` - For legal instruments (laws, regulations)

=== Legal Instruments

Legal instruments in `legal-instruments/` are stored as structured YAML files
with full text content organized hierarchically:

[source,yaml]
----
# yaml-language-server: $schema=../../schemas/jp-legal-instrument.yml
---
id: jp/diet-foreign-exchange-and-foreign-trade-act
title:
  - ja: 外国為替及び外国貿易法
  - en: Foreign Exchange and Foreign Trade Act (Act No. 228 of 1949)
url: https://www.japaneselawtranslation.go.jp/ja/laws/view/4857
type: jp/act-of-diet
authority: jp/diet
publisher: jp/diet
document_id: 昭和二十四年法律第二百二十八号
lang: ja
publish_date: '1949-12-01'
effective_date: '1949-12-01'

content:
  - type: chapter
    index: 1
    label: 第一章
    title: 総則
    content:
      - type: clause
        index: 1
        label: 第一条
        title: （目的）
        content:
          - この法律は、外国為替、外国貿易その他の対外取引が自由に行われることを基本とし...
----

Content types supported:

[cols="1,3"]
|===
| Type | Description

| `chapter`
| Chapter (章) with optional title and nested content

| `clause`
| Article/Clause (条) with label (e.g., "第一条")

| `numbered-list`
| Numbered list container (号)

| `list-item`
| Individual list item with label (e.g., "一", "イ")

|===

== Normalized Knowledge Graph Structure

The data follows a normalized structure where:

* **Entities are LIST-AGNOSTIC** - A person or organization can appear on multiple lists
* **Lists define sanctions regimes** - Each list has its own metadata
* **Entries are junction records** - They link entities to lists with list-specific data

=== IRI Structure

All resources use normalized IRIs:

```
# Entity (list-agnostic)
https://www.ammitto.org/entity/jp/{local_id}

# Entry (list-specific)
https://www.ammitto.org/entry/jp/{list_type}/{local_id}

# List
https://www.ammitto.org/list/jp/{list_type}
```

Example IRIs:

```
https://www.ammitto.org/entity/jp/al-qaida
https://www.ammitto.org/entry/jp/fefta-list/al-qaida
https://www.ammitto.org/list/jp/fefta-list
```

== Source Data Format

The source YAML files in `sources/` follow the announcement schema:

=== Announcement Schema

Standard sanction announcements with `sanction_details`:

[source,yaml]
----
announcement:
  title:
    - ja: 我が国の平和及び安全の維持のための措置に関するリスト
    - en: List concerning Measures for the Maintenance of Peace and Security of Japan
  url: https://www.meti.go.jp/policy/anpo/index.html
  publish_date: '2025-01-31'
  authority: jp/meti
  type: jp/fefta-end-user-list-announcement

sanction_details:
  instruments:
    - id: jp/diet-foreign-exchange-and-foreign-trade-act
    - id: jp/cabinet-order-foreign-exchange
  entities:
    - name:
        ja: アルカイダ
        en: Al Qa'ida
      type: organization
      effective_date: '2025-01-31'
      sanction_list: jp/fefta-end-user-list
      country_code: AF
      country_name:
        ja: アフガニスタン
        en: Islamic Republic of Afghanistan
      reason:
        - ja: 化学兵器の開発・拡散に関与
          en: Involved in development or proliferation of chemical weapons
      measures:
        - type:
          - export_license_requirement
          ja: 輸出に際して許可が必要
          en: Export license required
      aliases:
        - Al Qaeda
        - Islamic Salvation Foundation
        - The Base
----

=== Field Definitions

[cols="1,4"]
|===
| Field | Description

| `announcement.title`
| Bilingual title array with `ja` and `en` entries

| `announcement.publish_date`
| Date of publication (YYYY-MM-DD)

| `announcement.authority`
| Issuing authority ID (e.g., jp/meti, jp/mof)

| `sanction_details.instruments`
| Legal instruments providing authority

| `sanction_details.entities`
| Array of sanctioned entities

| `entity.name.ja`
| Japanese name

| `entity.name.en`
| English name

| `entity.type`
| Entity type: individual, organization, vessel, aircraft

| `entity.country_code`
| ISO 3166-1 alpha-2 country code (e.g., AF, CN, IR)

| `entity.country_name`
| Bilingual country name with `ja` and `en` entries

| `entity.sanction_list`
| Sanction list ID reference (e.g., `jp/fefta-end-user-list`)

| `entity.aliases`
| Array of alternative names

| `entity.measures`
| Sanction measures imposed

| `entity.reason`
| Array of bilingual reasons for sanction

|===

== Updating Data

=== METI Foreign User List (外国ユーザーリスト)

The Foreign User List is fully automated. Simply run:

[source,shell]
----
ruby scripts/update_foreign_user_list.rb
----

This script will:
1. Download the latest Excel file from the METI website
2. Convert it to YAML format
3. Save to `sources/sanction-lists/foreign-user-list/YYYYMMDD.yml`

The entity IDs follow the format `jp.meti.ful.{id}` where `{id}` is the original list number.

=== FEFTA End-User List (Manual Process)

For the FEFTA End-User List published as PDF:

. **Download the PDF** from the METI website
   * URL: https://www.meti.go.jp/policy/anpo/index.html
   * Look for the latest end-user list publication

. **Convert PDF to HTML** using Adobe Acrobat
   * Open the PDF in Adobe Acrobat
   * Export to HTML (not "HTML with embedded images")
   * Save to `reference-docs/YYYYMMDD-jp-end-user-list.html`

. **Fix table structure** in the HTML file
   * Tables spanning multiple pages need to be joined
   * Ensure all entity rows are in a single table

. **Run the conversion script**
+
[source,shell]
----
ruby scripts/convert_html_to_yaml.rb reference-docs/YYYYMMDD-jp-end-user-list.html sources/sanction-lists/fefta-list/YYYYMMDD.yml
----

=== Russia Sanctions List (ロシア制裁リスト)

For Russia sanctions related to the Ukraine situation:

. **Download the PDF** from the METI website
   * URL: https://www.meti.go.jp/policy/external_economy/trade_control/01_seido/04_seisai/crimea.html
   * Look for "輸出等に係る禁止措置の対象となるロシア連邦の特定団体"
   * Save to `reference-docs/russia-sanctions/YYYYMMDD.pdf`

. **Run the PDF parser script**
+
[source,shell]
----
ruby scripts/parse_russia_sanctions_pdf.rb
----

This will generate YAML files in `sources/sanction-lists/russia-sanctions/YYYYMMDD.yml`.

=== Legal Instruments

To convert XML legal instruments to YAML:

[source,shell]
----
ruby scripts/convert_xml_to_yaml.rb reference-docs/324AC0000000228_20250601_504AC0000000068.xml sources/legal-instruments/diet-foreign-exchange-and-foreign-trade-act.yml
----
